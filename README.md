# Git Lesson


## リモートリポジトリとローカルリポジトリとはそれぞれなんでしょうか？

- リモートリポジトリ
  - ネット上に配置して複数人で共有するためのリポジトリ。
- ローカルリポジトリ
  - 開発者一人ひとりが使用するために自分のPC上に配置するためのリポジトリ。

## プッシュとマージの違いは何でしょうか？

- プッシュ
  - ローカルの内容をリモートにアップする作業のこと。
- マージ
  - masterブランチにその変更履歴を取り込んで変更を反映させること。

## コミットとプッシュの違い

- コミット
  - ローカル上の履歴に、変更内容を「保存・確定」する作業。
- プッシュ
  - ローカルで確定した履歴を、リモートへ「送信・反映」する作業。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

一目で変更内容がわかるように書く。
各チームやプロジェクトでのルールに準拠したメッセージにする。



## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

他の人の確認が入るか否か。
プルリクエストにてコードレビューをしてもらうことで、間違いやバグなどをそのまま本番環境に反映することを未然に防げる。



## コンフリクトを起こしてしまった場合、どう対処すべきですか？

3通りの方法があり、
1. 先にマージされた変更内容を取り込む
2. 後にマージしようとしている変更内容を取り込む
3. どちらの変更内容も取り込む
